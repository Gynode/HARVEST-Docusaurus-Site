Getting Started​

Welcome to the HARVEST blockchain ecosystem! This comprehensive user guide will help you navigate all the features and capabilities of the HARVEST platform, from basic wallet operations to advanced agricultural applications. Whether you're a farmer, agricultural business, environmental organization, or technology enthusiast, this guide provides step-by-step instructions for using HARVEST blockchain effectively.

What is HARVEST Blockchain?​

HARVEST is a specialized blockchain platform built as a Cardano sidechain, designed specifically for agricultural and environmental applications. The platform enables transparent supply chain tracking, efficient agricultural commerce, environmental asset management, and innovative financial services tailored to the agricultural sector.

The HARVEST ecosystem includes several key components that work together to provide comprehensive blockchain functionality:

  • HRV Token: The native cryptocurrency with a total supply of 1 billion tokens, used for all transactions and network governance
  • NFT System: Support for up to 3,125 unique non-fungible tokens representing agricultural assets, carbon credits, and digital collectibles
  • Smart Contracts: Programmable functionality for complex agricultural applications including supply chain tracking and insurance
  • Interoperability: Seamless integration with Cardano mainnet through a secure two-way peg mechanism

System Requirements​

To use HARVEST blockchain applications, you'll need:

  • Computer: Windows 10+, macOS 10.14+, or Linux (Ubuntu 18.04+ recommended)
  • Internet Connection: Stable broadband connection for blockchain synchronization
  • Storage: At least 10 GB free space for full node operation (light wallets require minimal storage)
  • Memory: 4 GB RAM minimum, 8 GB recommended for optimal performance
  • Browser: Modern web browser (Chrome, Firefox, Safari, or Edge) for web-based applications

Creating Your First HARVEST Address​

Your HARVEST address serves as your unique identifier on the blockchain and enables you to receive HRV tokens and NFTs. Follow these steps to create your first address:

  1. Download the HARVEST Wallet: Visit the official HARVEST website and download the wallet application for your operating system
  2. Install and Launch: Follow the installation instructions and launch the wallet application
  3. Create New Wallet: Select "Create New Wallet" and follow the setup wizard
  4. Secure Your Seed Phrase: Write down the 12-word recovery phrase and store it securely offline
  5. Set Password: Choose a strong password to protect your wallet file
  6. Generate Address: Your first HARVEST address will be automatically generated

Important Security Note: Your seed phrase is the master key to your wallet. Never share it with anyone and store it securely offline. Anyone with access to your seed phrase can control your funds.

Understanding HARVEST Addresses​

HARVEST addresses are unique identifiers that enable you to send and receive tokens and NFTs on the blockchain. Addresses are derived from cryptographic keys and provide several important properties:

  • Uniqueness: Each address is mathematically unique and cannot be duplicated
  • Privacy: Addresses don't reveal personal information about their owners
  • Verification: All transactions to and from addresses are cryptographically verified
  • Compatibility: HARVEST addresses are compatible with Cardano address formats

You can create multiple addresses within a single wallet for different purposes such as business transactions, personal use, or specific agricultural projects. Each address maintains its own balance and transaction history while being controlled by the same wallet.

HARVEST Wallet Guide​

The HARVEST wallet is your primary interface for interacting with the blockchain, managing your HRV tokens and NFTs, and accessing advanced features like staking and marketplace trading. This section provides comprehensive instructions for using all wallet features effectively.

Wallet Installation and Setup​

The HARVEST wallet is available in multiple formats to accommodate different user preferences and technical requirements:

Desktop Application: Full-featured wallet with complete blockchain functionality, suitable for regular users and advanced operations. The desktop wallet includes built-in node functionality and can operate independently without relying on external services.

Web Application: Browser-based wallet that provides convenient access to core features without requiring software installation. The web wallet connects to HARVEST network nodes and provides most functionality while maintaining ease of use.

Mobile Application: Lightweight wallet optimized for mobile devices, providing essential features for everyday use. The mobile wallet includes QR code scanning for easy address sharing and transaction creation.

Command Line Interface: Advanced wallet for developers and technical users who prefer command-line operations. The CLI wallet provides complete access to all blockchain features and can be integrated into automated systems.

Wallet Interface Overview​

The HARVEST wallet interface is designed to be intuitive while providing access to powerful blockchain functionality. The main interface includes several key sections:

Dashboard: Provides an overview of your wallet status including total HRV balance, NFT count, staking information, and recent transaction activity. The dashboard also displays current network status and any important announcements.

Send/Receive: Enables you to send HRV tokens or NFTs to other addresses and generate receiving addresses for incoming transactions. This section includes address book functionality for storing frequently used addresses and transaction templates for common operations.

Transaction History: Displays a complete record of all transactions associated with your wallet addresses. The history includes detailed information about each transaction including amounts, fees, timestamps, and confirmation status.

NFT Gallery: Showcases all NFTs owned by your wallet addresses with rich metadata display and management options. The gallery includes filtering and sorting capabilities to help you organize large NFT collections.

Staking: Provides access to staking functionality including delegation options, reward tracking, and unstaking procedures. This section displays current staking status and projected rewards based on network conditions.

Settings: Enables customization of wallet behavior including network selection, fee preferences, security settings, and backup options. The settings section also includes tools for wallet maintenance and troubleshooting.

Creating and Managing Addresses​

The HARVEST wallet supports multiple addresses within a single wallet, enabling you to organize your funds and activities according to your needs. Each address maintains its own balance and transaction history while being controlled by the same wallet keys.

To create a new address:

  1. Open Address Management: Navigate to the "Addresses" section in your wallet
  2. Generate New Address: Click "Create New Address" and provide a descriptive label
  3. Verify Address: Confirm that the new address is displayed correctly in your address list
  4. Share Address: Use the "Copy" or "QR Code" functions to share your address for receiving funds

Address labels help you organize different addresses for specific purposes such as:

  • Personal: For personal transactions and everyday use
  • Business: For agricultural business operations and commercial transactions
  • Staking: Dedicated address for staking operations and reward collection
  • NFT Trading: Specialized address for NFT marketplace activities
  • Project-Specific: Addresses for specific agricultural projects or initiatives

Backup and Recovery​

Proper backup procedures are essential for protecting your HARVEST assets. The wallet provides multiple backup options to ensure you can recover your funds even if your device is lost or damaged.

Seed Phrase Backup: The most important backup is your 12-word seed phrase, which can restore your entire wallet on any compatible device. Write down your seed phrase on paper and store it in a secure location away from your computer. Never store your seed phrase digitally or share it with anyone.

Wallet File Backup: The wallet creates encrypted backup files that contain your wallet configuration and address labels. These files are protected by your wallet password and can be used to restore your wallet settings on the same device.

Address Export: You can export individual addresses and their private keys for specific backup purposes. This option is useful for creating cold storage solutions or sharing specific addresses with trusted parties.

To restore your wallet from backup:

  1. Install Wallet Software: Download and install the HARVEST wallet on your new device
  2. Select Restore Option: Choose "Restore from Seed Phrase" during wallet setup
  3. Enter Seed Phrase: Carefully enter your 12-word seed phrase in the correct order
  4. Set New Password: Choose a new password for your restored wallet
  5. Synchronize: Allow the wallet to synchronize with the blockchain to restore your transaction history

Security Features​

The HARVEST wallet includes comprehensive security features designed to protect your assets while maintaining ease of use:

Encryption: All wallet data is encrypted using industry-standard encryption algorithms. Your private keys are never stored in plain text and are protected by your wallet password.

Multi-Signature Support: The wallet supports multi-signature addresses that require multiple signatures to authorize transactions. This feature is useful for business accounts and shared funds that require multiple approvals.

Hardware Wallet Integration: The wallet can integrate with popular hardware wallets for enhanced security. Hardware wallets store your private keys on dedicated devices that never connect to the internet.

Transaction Verification: The wallet provides detailed transaction previews that show exactly what will happen before you confirm any transaction. Always verify transaction details before confirming.

Automatic Locking: The wallet automatically locks after a period of inactivity to prevent unauthorized access. You can customize the auto-lock timeout in the security settings.

Backup Reminders: The wallet periodically reminds you to verify your backup procedures and update your security settings. Regular security maintenance helps protect your assets over time.

HRV Token Management​

HRV tokens serve as the native currency of the HARVEST blockchain, enabling all economic activities within the ecosystem. This section provides comprehensive guidance for managing your HRV tokens effectively, including sending and receiving transactions, understanding fees, and optimizing your token usage.

Understanding HRV Tokens​

HRV (HARVEST) tokens are the fundamental unit of value on the HARVEST blockchain, with a total supply of 1 billion tokens. HRV tokens serve multiple purposes within the ecosystem:

Transaction Fees: All blockchain transactions require HRV tokens to pay network fees. These fees compensate validators for processing transactions and maintaining network security.

Staking: HRV tokens can be staked to participate in network consensus and earn rewards. Staking helps secure the network while providing passive income for token holders.

Governance: HRV token holders can participate in network governance by voting on protocol upgrades, parameter changes, and ecosystem development proposals.

Commerce: HRV tokens facilitate agricultural commerce including payments for goods and services, supply chain financing, and insurance premiums.

Marketplace: HRV tokens are used for all marketplace transactions including NFT purchases, auction bids, and trading fees.

Sending HRV Tokens​

Sending HRV tokens is a straightforward process that enables you to make payments, transfer funds between your addresses, or send tokens to other users. Follow these steps to send HRV tokens:

  1. Open Send Interface: Navigate to the "Send" section in your wallet
  2. Enter Recipient Address: Input the destination address or select from your address book
  3. Specify Amount: Enter the amount of HRV tokens to send
  4. Review Transaction: Verify the recipient address, amount, and estimated fee
  5. Confirm Transaction: Enter your wallet password and confirm the transaction
  6. Monitor Status: Track the transaction status until it receives blockchain confirmation

Important Considerations:

  • Address Verification: Always double-check recipient addresses before confirming transactions. Blockchain transactions are irreversible.
  • Fee Estimation: The wallet automatically calculates appropriate fees based on network conditions. Higher fees result in faster confirmation times.
  • Balance Requirements: Ensure you have sufficient HRV tokens to cover both the transfer amount and transaction fees.
  • Confirmation Times: Transactions typically confirm within 1-3 minutes under normal network conditions.

Receiving HRV Tokens​

Receiving HRV tokens requires sharing your HARVEST address with the sender. Your wallet provides several convenient methods for sharing addresses:

Address Copying: Copy your address to the clipboard for pasting into messages or forms. Always verify that the copied address is correct before sharing.

QR Code Generation: Generate QR codes that can be scanned by mobile wallets for easy address entry. QR codes reduce the risk of address entry errors.

Payment Requests: Create payment requests that include your address, requested amount, and optional message. Payment requests can be shared via email, messaging apps, or social media.

Address Labels: Use descriptive labels for different addresses to help senders understand the purpose of each payment. Labels are only visible in your wallet and don't affect blockchain operations.

Transaction Fees and Optimization​

HARVEST blockchain transaction fees are designed to be low and predictable while ensuring network sustainability. Understanding fee structures helps you optimize your transaction costs:

Base Fees: All transactions include a base fee that covers basic processing costs. Base fees are fixed and don't vary with transaction amount.

Size-Based Fees: Transactions with large amounts of data (such as complex smart contracts or detailed metadata) incur additional fees based on data size.

Priority Fees: Optional priority fees can be added to expedite transaction processing during periods of high network activity.

Batch Transactions: Sending multiple payments in a single transaction reduces per-payment costs compared to individual transactions.

To minimize transaction fees:

  • Consolidate Payments: Combine multiple payments into batch transactions when possible
  • Optimize Timing: Send transactions during periods of low network activity for lower fees
  • Use Appropriate Priority: Only use priority fees when fast confirmation is essential
  • Monitor Network Status: Check network congestion before sending large or time-sensitive transactions

Advanced Token Features​

The HARVEST wallet includes several advanced features for sophisticated token management:

Multi-Send: Send HRV tokens to multiple recipients in a single transaction, reducing fees and simplifying bulk payments. This feature is particularly useful for payroll, dividend distributions, and supplier payments.

Scheduled Transactions: Create transactions that execute automatically at specified times or when certain conditions are met. Scheduled transactions enable automated payments and recurring transfers.

Escrow Services: Use built-in escrow functionality for secure transactions with untrusted parties. Escrow services hold funds until specified conditions are met, providing protection for both buyers and sellers.

Token Splitting: Divide large token amounts across multiple addresses for improved privacy and security. Token splitting can also help optimize staking strategies and reduce risk exposure.

Cold Storage Integration: Transfer tokens to offline storage addresses for enhanced security. Cold storage protects large token holdings from online threats while maintaining the ability to move funds when needed.

NFT Features and Management​

The HARVEST blockchain's NFT (Non-Fungible Token) system provides comprehensive functionality for creating, managing, and trading unique digital assets that represent agricultural products, environmental credits, land ownership, and digital collectibles. With support for up to 3,125 unique NFTs, the system is designed to serve the diverse needs of the agricultural community while maintaining ease of use and powerful functionality.

Understanding HARVEST NFTs​

HARVEST NFTs are unique digital tokens that represent ownership of specific assets or rights within the agricultural ecosystem. Unlike fungible tokens such as HRV that are interchangeable, each NFT has distinct characteristics and cannot be replicated or substituted. HARVEST NFTs serve several important purposes:

Agricultural Asset Representation: NFTs can represent physical agricultural assets such as livestock, equipment, land parcels, or crop yields. These NFTs include detailed metadata about the physical assets and enable digital ownership tracking.

Supply Chain Documentation: NFTs serve as digital certificates that document the journey of agricultural products from farm to consumer. Each NFT contains immutable records of production methods, quality certifications, and handling procedures.

Environmental Credits: NFTs represent environmental assets such as carbon credits, biodiversity certificates, and water quality improvements. These NFTs enable transparent trading of environmental benefits and support sustainable agricultural practices.

Digital Collectibles: NFTs can represent digital art, commemorative items, and collectibles related to agricultural themes. These NFTs support the agricultural community while providing entertainment and investment opportunities.

Ownership Certificates: NFTs serve as digital certificates for land ownership, equipment titles, and other legal documents. These certificates provide tamper-proof records that can be easily verified and transferred.

NFT Metadata and Attributes​

HARVEST NFTs include rich metadata structures that provide comprehensive information about the represented assets. The metadata system supports both standardized fields for interoperability and custom fields for specific agricultural applications:

Basic Information: All NFTs include fundamental information such as name, description, creation date, and creator identity. This information provides basic identification and provenance for each NFT.

Agricultural Attributes: NFTs representing agricultural assets include specialized attributes such as crop type, harvest date, geographic coordinates, farming methods, and quality certifications. These attributes enable detailed tracking and verification of agricultural products.

Environmental Data: NFTs representing environmental assets include attributes such as carbon sequestration amounts, biodiversity impact measurements, and sustainability certifications. This data enables accurate valuation and trading of environmental benefits.

Ownership History: All NFTs maintain complete ownership history including previous owners, transfer dates, and transaction details. This history provides transparency and helps establish provenance for valuable assets.

Multimedia Content: NFTs can include images, videos, documents, and other multimedia content that provides visual representation and additional documentation. Multimedia content is stored using distributed storage systems with cryptographic verification.

Creating and Minting NFTs​

The HARVEST platform provides multiple methods for creating and minting NFTs, accommodating different user needs and technical capabilities:

Individual Minting: Create single NFTs through the wallet interface by providing metadata, uploading multimedia content, and specifying ownership details. Individual minting is suitable for unique assets and small-scale operations.

Batch Minting: Create multiple NFTs simultaneously using standardized templates and bulk data import. Batch minting is efficient for large collections and repetitive asset types such as carbon credit certificates.

Automated Minting: Integrate with external systems to automatically create NFTs based on real-world events such as harvest completion, quality certification, or environmental monitoring. Automated minting reduces manual effort and ensures timely NFT creation.

To mint an NFT using the wallet interface:

  1. Access NFT Section: Navigate to the "NFTs" section in your HARVEST wallet
  2. Select Create NFT: Click "Create New NFT" to open the minting interface
  3. Enter Basic Information: Provide name, description, and category for your NFT
  4. Add Metadata: Fill in relevant attributes such as agricultural data, environmental information, or ownership details
  5. Upload Content: Add images, documents, or other multimedia content to your NFT
  6. Set Royalties: Configure royalty percentages for future sales (optional)
  7. Review and Confirm: Verify all information and confirm the minting transaction
  8. Pay Fees: Complete the transaction by paying the required HRV token fees

Minting Considerations:

  • Uniqueness: Ensure your NFT represents a truly unique asset or has distinct characteristics
  • Metadata Accuracy: Provide accurate and complete metadata to maximize NFT value and utility
  • Content Quality: Use high-quality images and documents to enhance NFT presentation
  • Royalty Strategy: Consider appropriate royalty percentages that balance creator compensation with market appeal

NFT Ownership and Transfers​

NFT ownership on the HARVEST blockchain is cryptographically secured and easily verifiable. The ownership system provides several important features:

Proof of Ownership: The blockchain provides immutable proof of NFT ownership that cannot be forged or disputed. Ownership records are publicly verifiable while maintaining user privacy.

Transfer Mechanisms: NFTs can be transferred through direct transfers, marketplace sales, auctions, or smart contract operations. All transfers are recorded on the blockchain and update ownership records automatically.

Approval Systems: NFT owners can approve third parties to transfer specific NFTs or all NFTs in their collection. Approval systems enable marketplace operations and automated trading without compromising security.

Batch Operations: Multiple NFTs can be transferred simultaneously to reduce transaction costs and simplify portfolio management. Batch operations are particularly useful for large collections and business operations.

To transfer an NFT:

  1. Select NFT: Choose the NFT you want to transfer from your collection
  2. Enter Recipient: Specify the destination address or select from your address book
  3. Review Transfer: Verify the NFT details and recipient address
  4. Confirm Transaction: Authorize the transfer and pay the required fees
  5. Monitor Status: Track the transaction until it receives blockchain confirmation

Transfer Considerations:

  • Address Verification: Always verify recipient addresses before confirming transfers
  • Ownership Rights: Ensure you have the right to transfer the NFT and understand any restrictions
  • Tax Implications: Consider potential tax consequences of NFT transfers in your jurisdiction
  • Record Keeping: Maintain records of NFT transfers for accounting and tax purposes

NFT Marketplace Integration​

The HARVEST NFT marketplace provides a decentralized platform for buying, selling, and trading NFTs within the ecosystem. The marketplace includes several trading mechanisms:

Fixed Price Sales: List NFTs at predetermined prices with immediate purchase options. Fixed price sales provide certainty for both buyers and sellers and enable quick transactions.

Auction Systems: Sell NFTs through competitive bidding processes that can maximize sale prices for valuable or rare items. The marketplace supports both English auctions (ascending bids) and Dutch auctions (descending prices).

Bundle Sales: Sell multiple related NFTs together as a package deal. Bundle sales are useful for collections, supply chain documentation sets, or complementary agricultural assets.

Fractional Ownership: Enable multiple parties to own shares of high-value NFTs through fractional ownership mechanisms. Fractional ownership makes expensive agricultural assets accessible to smaller investors.

To list an NFT for sale:

  1. Access Marketplace: Navigate to the marketplace section in your wallet or use the web marketplace interface
  2. Select NFT: Choose the NFT you want to sell from your collection
  3. Choose Sale Type: Select fixed price, auction, or bundle sale format
  4. Set Parameters: Specify price, duration, and any special conditions
  5. Review Listing: Verify all listing details and marketplace fees
  6. Confirm Listing: Authorize the listing transaction and pay required fees
  7. Monitor Activity: Track views, bids, and other marketplace activity for your listing

Marketplace Best Practices:

  • Competitive Pricing: Research similar NFTs to set competitive prices
  • Quality Presentation: Use high-quality images and detailed descriptions to attract buyers
  • Timing Strategy: Consider market conditions and seasonal factors when listing agricultural NFTs
  • Communication: Respond promptly to buyer inquiries and provide additional information when requested

Advanced NFT Features​

The HARVEST NFT system includes several advanced features that provide additional functionality for sophisticated use cases:

Programmable Royalties: NFTs can include smart contract functionality that automatically distributes royalty payments to multiple beneficiaries. Programmable royalties enable complex revenue sharing arrangements for agricultural cooperatives and environmental projects.

Conditional Transfers: NFTs can include conditions that must be met before transfers are allowed. Conditional transfers enable sophisticated ownership arrangements such as lease agreements and performance-based transfers.

Metadata Updates: Authorized parties can update NFT metadata to reflect changes in the underlying assets. Metadata updates enable dynamic NFTs that evolve over time, such as supply chain documentation that updates as products move through the value chain.

Staking Integration: NFTs can be staked to earn rewards or participate in governance decisions. NFT staking provides additional utility and income opportunities for NFT holders.

Cross-Chain Compatibility: HARVEST NFTs can be transferred to Cardano mainnet through the interoperability bridge. Cross-chain compatibility enables access to broader NFT markets and applications.

NFT Portfolio Management​

Effective NFT portfolio management helps you organize, track, and optimize your NFT investments and holdings:

Collection Organization: Use categories, tags, and custom labels to organize your NFT collection. Organization systems help you quickly find specific NFTs and track different types of assets.

Valuation Tracking: Monitor the market value of your NFTs using marketplace data and price history. Valuation tracking helps you make informed decisions about buying, selling, and holding NFTs.

Performance Analytics: Track the performance of your NFT investments including purchase prices, current values, and realized gains or losses. Performance analytics help you optimize your investment strategy.

Risk Management: Diversify your NFT portfolio across different categories, creators, and risk levels. Risk management helps protect your investment from market volatility and specific asset risks.

Tax Reporting: Maintain detailed records of NFT transactions for tax reporting purposes. Many jurisdictions require reporting of NFT gains and losses, and proper record keeping simplifies compliance.

The HARVEST wallet includes portfolio management tools that provide:

  • Collection Overview: Visual display of your entire NFT collection with filtering and sorting options
  • Value Tracking: Real-time valuation of your NFT holdings based on marketplace data
  • Transaction History: Complete record of all NFT purchases, sales, and transfers
  • Performance Reports: Detailed analytics on portfolio performance and investment returns
  • Export Functions: Export portfolio data for external analysis and tax reporting
